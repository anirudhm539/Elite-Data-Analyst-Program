# ADF Control Flow Activities

## Purpose
Control pipeline execution logic.

## Activities

### If Condition
- True/False branching

### ForEach
- Loop through items

### Switch
- Multiple branches

### Until
- Repeat until condition met

### Validation
- Check file/folder existence

### Wait
- Pause execution

### Fail
- Mark pipeline failed

### Execute Pipeline
- Run another pipeline

## Enterprise Practices
- Use validation before copy
- Use parent-child architecture
- Keep pipelines modular
- Use loops instead of duplication

## Interview Notes
- ForEach
- Switch vs If
- Execute Pipeline
- Parent Child Architecture# ADF Control Flow Activities
